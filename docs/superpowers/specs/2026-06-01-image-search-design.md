# Image Search First Phase Design

Date: 2026-06-01

## Goal

Build the first complete production-oriented image search module for the Shopify app. The feature runs on the storefront, opens from a floating widget, lets shoppers upload an image, returns visually similar products, supports favorites and non-navigating add-to-cart, and adds a Similar Products section on Shopify native product detail pages.

The implementation uses a real interface path rather than a UI-only mock:

- Storefront Theme App Extension for the floating image search modal and product detail recommendations.
- Shopify React Router app backend for search, indexing, favorites, upload history, and recommendation APIs.
- Independent Python FastAPI embedding service using `openai/clip-vit-base-patch16`.
- Milvus collection with 512-dimensional vectors.
- PostgreSQL as the business data store.

## User-Facing Scope

### Floating Image Search Modal

The storefront shows a floating entry point. Clicking it opens an `Image Search` modal.

The first modal screen includes:

- Uploaded image preview.
- `Upload New Image` action.
- Recent upload thumbnails.
- Favorite products entry and visible favorite state on result cards.
- `Available Products Only` filter.
- `Sort by: Most Relevant` control.
- Product result grid.
- Product card image, title, price, variant or size label, favorite button, and `Add to Cart`.

Interactions:

- Uploading an image runs image search and refreshes the result grid.
- Clicking a product image or title navigates to Shopify native product detail page `/products/{handle}`.
- Clicking `Add to Cart` calls Shopify Ajax Cart API `/cart/add.js`.
- Add-to-cart does not navigate, close the modal, or clear search results.
- Successful add-to-cart changes the button state to `Added`.
- Favorite toggles update local UI immediately and sync to the backend when possible.
- Recent uploads remain visible after a failed search.

### Product Detail Similar Products

Shopify native product pages include a `Similar Products` section below the primary product content.

The section:

- Uses the current product's indexed primary image vector.
- Queries Milvus for visually similar products.
- Excludes the current product.
- Shows horizontal product cards.
- Reuses favorite and Ajax add-to-cart behavior.
- Navigates cards to `/products/{handle}`.

The first phase does not build outfit or bundle recommendations. It only ships visual Similar Products.

## Non-Goals

The first phase does not include:

- Text search.
- Smart Cart.
- Chat-based shopping assistant.
- Outfit or complete-the-look recommendation logic.
- Review evidence or AI explanation snippets.
- A/B testing dashboards.
- Manual image crop or box selection.
- Replacing Shopify native product detail pages.

## Architecture

```txt
Storefront Theme App Extension
  -> Shopify App Backend APIs
  -> FastAPI CLIP Embedding Service
  -> Milvus vector search
  -> PostgreSQL product and user-state data
  -> Storefront UI response
```

Responsibilities:

- Theme App Extension owns storefront UI and Shopify Ajax Cart calls.
- Shopify backend owns API orchestration, shop isolation, indexing, persistence, and response shaping.
- FastAPI embedding service owns image-to-vector conversion.
- Milvus owns nearest-neighbor vector search.
- PostgreSQL owns product metadata, upload history, favorites, and index status.

## Frontend Design

### Theme App Extension Blocks

Create a Theme App Extension with two storefront surfaces:

- App embed for the floating image search widget on storefront pages.
- Product app block for the Similar Products section on product detail pages.

The floating widget should be visually close to the provided reference:

- Modal header with `Image Search` title and close button.
- Left panel for upload preview and recent images.
- Right panel for filters and product results.
- Scrollable product grid.
- Compact product cards optimized for repeated browsing.

### State Model

The storefront script manages:

- Modal open state.
- Selected upload image.
- Search loading state.
- Search results.
- Add-to-cart state per variant.
- Favorite state per product.
- Recent upload thumbnails.
- Error and empty states.

Use `localStorage` for immediate anonymous shopper continuity:

- `lensCartAi.anonymousId`
- `lensCartAi.recentUploads`
- `lensCartAi.favoriteProducts`

The backend stores the same state when it can identify `shop_domain` and anonymous browser id. If customer identity becomes available later, favorites and upload history can be merged by customer id without changing the UI contract.

## Backend API Design

All storefront APIs must isolate by shop domain. Requests from Theme App Extension include the current shop domain and an anonymous id generated by the storefront script.

### `POST /api/image-search`

Purpose: run image search from an uploaded shopper image.

Input:

- Image file.
- `shopDomain`.
- `anonymousId`.
- `availableOnly`.
- `sort`, initially `most_relevant`.
- `limit`, default 24.

Flow:

1. Validate file type and size.
2. Store upload metadata and thumbnail.
3. Call embedding service `/embed-image`.
4. Validate returned vector length is 512.
5. Query Milvus with `shop_domain` filter and optional `available_for_sale == true`.
6. Read matching product display data from PostgreSQL.
7. Filter missing or inactive records.
8. Return normalized product card data.

Output:

- `uploadId`
- `results`
- `favorites`
- `recentUploads`
- `queryMeta`

### `GET /api/products/:productId/similar`

Purpose: return visual Similar Products for a product detail page.

Input:

- `shopDomain`
- `productId` or Shopify product gid.
- `limit`, default 12.

Flow:

1. Resolve the current product in PostgreSQL.
2. Find its indexed primary image vector id.
3. Query Milvus using the existing product vector.
4. Exclude the current product.
5. Return normalized product card data.

This endpoint does not regenerate embeddings during page views.

### Favorites

`GET /api/favorites`

- Returns favorite products for `shopDomain` and `anonymousId`.

`POST /api/favorites`

- Adds a favorite product.
- Body includes `shopDomain`, `anonymousId`, and product gid.

`DELETE /api/favorites/:productId`

- Removes a favorite product.

### Upload History

`GET /api/upload-history`

- Returns recent upload thumbnails for `shopDomain` and `anonymousId`.

`POST /api/upload-history`

- Saves metadata for a successful upload.
- The search endpoint may call the same internal service, so the standalone endpoint is mainly for future direct sync.

### Product Indexing

`POST /api/admin/index-products`

Purpose: build or refresh the searchable product image index.

Flow:

1. Authenticate as Shopify admin app request.
2. Pull product, variant, price, availability, handle, and image data from Shopify Admin GraphQL.
3. For each eligible product image, call the embedding service.
4. Upsert product display records in PostgreSQL.
5. Upsert image embedding metadata in PostgreSQL.
6. Upsert vectors into Milvus.
7. Record per-product success or failure state.

Later phases can add webhooks for incremental indexing after product create, update, delete, or media update events.

## Embedding Service Design

Path:

```txt
services/embedding
```

Technology:

- Python FastAPI.
- PyTorch.
- Hugging Face Transformers.
- Model: `openai/clip-vit-base-patch16`.

API:

`POST /embed-image`

Input:

- Multipart image file, or image URL for indexing.

Output:

```json
{
  "model": "openai/clip-vit-base-patch16",
  "dimension": 512,
  "vector": [0.0]
}
```

The returned vector must be L2-normalized so Milvus similarity scores are stable.

Runtime configuration in the Shopify app:

```txt
EMBEDDING_PROVIDER=clip_http
EMBEDDING_API_URL=http://localhost:8001
EMBEDDING_DIMENSION=512
```

The Node app never loads PyTorch directly. It calls the embedding service over HTTP.

Use Prisma with PostgreSQL.

- `primaryVariantTitle`
- `createdAt`
- `updatedAt`

Core fields:

- `id`
- `shopDomain`
- `shopifyProductGid`

- `vectorId`
- `embeddingModel`
- `embeddingDimension`
- `isPrimary`
- `indexedAt`

- `shopDomain + shopifyMediaGid`

- `imageStorageKey`

### `FavoriteProduct`

Stores favorite product records.

Core fields:

- `id`
- `shopDomain`
- `anonymousId`
- `customerGid`

## Upload Storage

Production upload images and thumbnails should use S3-compatible object storage such as S3, Cloudflare R2, or MinIO.

Configuration:

```txt
UPLOAD_STORAGE_PROVIDER=s3
UPLOAD_STORAGE_BUCKET=
UPLOAD_STORAGE_ENDPOINT=
UPLOAD_STORAGE_ACCESS_KEY_ID=
UPLOAD_STORAGE_SECRET_ACCESS_KEY=
UPLOAD_STORAGE_PUBLIC_BASE_URL=
```

Local development may use filesystem storage under a gitignored `storage/uploads` directory through:

```txt
UPLOAD_STORAGE_PROVIDER=local
```

## Milvus Design

Collection name:

```txt
product_image_embeddings_512
```

Vector dimension:

```txt
512
```

Fields:

- `vector_id`
- `embedding`
- `shop_domain`
- `shopify_product_gid`
- `shopify_media_gid`
- `available_for_sale`
- `product_type`
- `status`
- `created_at_unix`

Search rules:

- Always filter by `shop_domain`.
- Apply `available_for_sale == true` when `Available Products Only` is enabled.
- Exclude the current product for Similar Products.
- Return more raw Milvus hits than needed, then let PostgreSQL filtering remove stale records.

Metric:

- Use inner product with L2-normalized vectors. This is equivalent to cosine similarity for normalized CLIP embeddings and must be used consistently for both indexing and search.

## Error Handling

Storefront:

- Invalid image type: show upload validation error.
- Image larger than 5MB: show file size error.
- Embedding service failure: show image recognition service unavailable message.
- Milvus failure: show similar product search failure message.
- No results: show empty result state in the modal.
- Add-to-cart failure: restore button and show inline error state.
- Similar Products empty result: hide the product-page section or show a compact empty state.

Backend:

- Reject unsupported content types before calling embedding.
- Reject embedding responses with dimension other than 512.
- Filter missing PostgreSQL product records from search results.
- Record product indexing failures with retryable status and failure reason.
- Log request ids for image search, indexing, and embedding calls.

## Testing Strategy

Backend tests:

- Embedding provider validates 512-dimensional output.
- Image search service handles Milvus hits and missing product records.
- Favorites APIs create, list, and delete records.
- Upload history APIs return recent records by shop and anonymous id.
- Similar products excludes the current product.

Embedding service tests:

- Valid image returns model name, dimension 512, and vector.
- Invalid image returns a controlled error.
- Vector norm is approximately 1 after normalization.

Frontend tests:

- Modal opens from floating widget.
- Upload shows loading and result grid.
- Product card click navigates to `/products/{handle}`.
- Add-to-cart calls `/cart/add.js` and does not navigate.
- Favorite toggle persists to localStorage and calls backend.
- Recent uploads render after a completed search.
- Error and empty states render inside the modal.

Integration verification:

- Start PostgreSQL, Milvus, embedding service, and Shopify app backend.
- Run product indexing against a development Shopify store.
- Upload a test image and verify similar products return.
- Open a product detail page and verify Similar Products renders.

## Deployment Notes

Local development startup order:

1. PostgreSQL.
2. Milvus.
3. `services/embedding` FastAPI service.
4. Shopify React Router app.
5. Theme App Extension preview.
6. Product indexing job.

Production deployment:

- Deploy the Shopify app backend and FastAPI embedding service as separate services.
- Keep `EMBEDDING_API_URL` private when possible.
- Use managed PostgreSQL.
- Use managed or self-hosted Milvus.
- Use S3-compatible object storage for upload history assets.
- Set Milvus collection dimension to 512 before indexing.

## Acceptance Criteria

The first phase is complete when:

- Storefront floating widget opens the Image Search modal.
- Modal includes upload image, recent images, favorites, result grid, and add-to-cart controls.
- Uploading an image calls the real FastAPI CLIP embedding service.
- Search uses Milvus 512-dimensional vectors and returns product data from PostgreSQL.
- Product card clicks navigate to Shopify native product pages.
- Add to Cart works through `/cart/add.js` without page navigation.
- Favorites can be toggled and revisited.
- Product detail pages show Similar Products based on indexed product image vectors.
- Product indexing can populate PostgreSQL and Milvus from Shopify Admin product data.
- Embedding, search, add-to-cart, favorites, upload history, and similar recommendation errors have visible controlled states.
